How to Use tinyproxy to Quickly Set Up an HTTP/HTTPS Proxy

Posted on Tue 07 April 2026 in Networking

If you want a lightweight proxy server that is easy to install and simple to manage, tinyproxy is a great option. It is a small, fast HTTP/HTTPS proxy daemon that works well for personal servers, development environments, and quick VPS setups.

In this post, you will learn:

  • what tinyproxy is
  • how to install it on Ubuntu
  • how to configure it, including username and password authentication
  • how to use it in a browser with ZeroOmega
  • how to use it in command-line applications
  • where to read more

What is tinyproxy?

tinyproxy is a lightweight forward proxy server. It is designed to be small, efficient, and easy to configure.

It supports:

  • HTTP proxying
  • HTTPS tunneling via CONNECT
  • access control by client IP
  • basic username/password authentication
  • simple logging and configuration

Because it is much smaller than full-featured proxy servers like Squid, it is a good fit when you want a quick and clean setup without too much overhead.

Install tinyproxy on Ubuntu

First, update your package list and install tinyproxy:

sudo apt update
sudo apt install tinyproxy -y
````

After installation, check whether the service is running:

```bash
sudo systemctl status tinyproxy

You can also enable it to start automatically on boot:

sudo systemctl enable tinyproxy

Configure tinyproxy

On Ubuntu, the main configuration file is usually located at:

/etc/tinyproxy/tinyproxy.conf

Before editing it, make a backup:

sudo cp /etc/tinyproxy/tinyproxy.conf /etc/tinyproxy/tinyproxy.conf.bak

Then open it with an editor, say nano:

sudo nano /etc/tinyproxy/tinyproxy.conf

Important Configuration Options

Here are some of the most useful settings:

  • Port — the port Tinyproxy listens on
  • Listen — optional IP address to bind to
  • Allow — allow connections only from specific client IP addresses
  • BasicAuth — set a username and password

Example Configuration

Below is a simple example configuration:

User tinyproxy
Group tinyproxy

Port 8888
Timeout 600

# Optional: bind to a specific interface
# Listen 0.0.0.0

# Only allow trusted client IPs
Allow 127.0.0.1
Allow YOUR_CLIENT_IP
# If you want to allow all clients, comment out the previous Allow lines.

# Username and password authentication
BasicAuth myuser mystrongpassword

After saving the file, restart Tinyproxy:

sudo systemctl restart tinyproxy
sudo systemctl status tinyproxy

Test the Proxy

You can test the proxy with curl.

Without authentication:

curl -x http://YOUR_SERVER_IP:8888 https://ipinfo.io

With authentication:

curl -x http://myuser:mystrongpassword@YOUR_SERVER_IP:8888 https://ipinfo.io

If everything is working, the result should show your server's public IP instead of your local machine's IP.

Use tinyproxy in a Browser with SwitchyOmega 3 (ZeroOmega)

SwitchyOmega 3 (ZeroOmega) is a browser extension that makes it easy to switch proxy profiles.

Steps

  1. Install SwitchyOmega 3 (ZeroOmega) in your browser.
  2. Open the SwitchyOmega 3 settings page.
  3. Create a new proxy profile.
  4. Set the proxy type to HTTP.

  5. Enter:

  6. Server: YOUR_SERVER_IP

  7. Port: 8888

  8. If authentication is enabled, enter:

  9. Username: myuser

  10. Password: mystrongpassword
  11. Save the profile.
  12. Activate that profile from the extension menu.

After that, visit a site that shows your public IP address to verify that your traffic is going through the proxy.

Use tinyproxy in Command-Line Applications

Many command-line tools support proxies through environment variables.

Set Proxy Environment Variables

export http_proxy="http://myuser:mystrongpassword@YOUR_SERVER_IP:8888"
export https_proxy="http://myuser:mystrongpassword@YOUR_SERVER_IP:8888"
export no_proxy="127.0.0.1,localhost"

To remove them later:

unset http_proxy
unset https_proxy
unset no_proxy

Use with curl

You can either rely on the environment variables above or specify the proxy directly:

curl -x http://myuser:mystrongpassword@YOUR_SERVER_IP:8888 https://example.com

Use with wget

Example:

wget -e use_proxy=yes \
     -e http_proxy=http://myuser:mystrongpassword@YOUR_SERVER_IP:8888 \
     https://example.com/file.txt

Use with git

Configure Git to use the proxy globally:

git config --global http.proxy http://myuser:mystrongpassword@YOUR_SERVER_IP:8888
git config --global https.proxy http://myuser:mystrongpassword@YOUR_SERVER_IP:8888

To remove the proxy settings later:

git config --global --unset http.proxy
git config --global --unset https.proxy

Troubleshooting

1. Tinyproxy does not start

Check service status:

sudo systemctl status tinyproxy

Check whether the port is already in use:

ss -tulpen | grep 8888

2. 407 Proxy Authentication Required

This usually means:

  • the username or password is wrong
  • the client is not sending credentials
  • the browser or CLI tool is not configured correctly

3. Browser works, but CLI does not

Some command-line tools do not automatically use shell proxy variables. They may require their own proxy settings, such as git.

Security Tips

  • Restrict access using Allow
  • Use a strong password
  • Avoid exposing the proxy to the public internet without IP restrictions
  • Monitor logs if the proxy is reachable from outside
  • Consider firewall rules in addition to Tinyproxy configuration
